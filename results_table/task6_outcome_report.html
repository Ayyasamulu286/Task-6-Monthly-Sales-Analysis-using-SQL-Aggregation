🗃️ SQL Script (SQLite)

-- QUERY 1: Monthly Revenue & Order Volume
SELECT
    strftime('%Y', order_date)  AS year,
    strftime('%m', order_date)  AS month,
    SUM(amount)                 AS total_revenue,
    COUNT(DISTINCT order_id)    AS order_volume
FROM online_sales
GROUP BY year, month
ORDER BY year, month;

-- QUERY 2: Top 3 months by revenue
SELECT
    strftime('%Y', order_date)  AS year,
    strftime('%m', order_date)  AS month,
    SUM(amount)                 AS total_revenue,
    COUNT(DISTINCT order_id)    AS order_volume
FROM online_sales
GROUP BY year, month
ORDER BY total_revenue DESC
LIMIT 3;

-- QUERY 3: Revenue by product per month
SELECT
    strftime('%Y-%m', order_date) AS year_month,
    product_id,
    SUM(amount)                  AS product_revenue,
    COUNT(DISTINCT order_id)     AS order_count
FROM online_sales
GROUP BY year_month, product_id
ORDER BY year_month, product_revenue DESC;

-- QUERY 4: Cumulative revenue (window function)
SELECT
    strftime('%Y-%m', order_date)                                      AS year_month,
    SUM(amount)                                                        AS monthly_revenue,
    SUM(SUM(amount)) OVER (ORDER BY strftime('%Y-%m', order_date))    AS cumulative_revenue
FROM online_sales
GROUP BY year_month
ORDER BY year_month;

-- QUERY 5: NULL-safe aggregation using COALESCE
SELECT
    strftime('%Y-%m', order_date)   AS year_month,
    SUM(COALESCE(amount, 0))        AS total_revenue,
    COUNT(DISTINCT order_id)       AS order_volume
FROM online_sales
GROUP BY year_month
ORDER BY year_month;

🎤 Interview Questions – Answers

1. How do you group data by month and year?

Use GROUP BY EXTRACT(YEAR FROM order_date), EXTRACT(MONTH FROM order_date) in PostgreSQL, or strftime('%Y', order_date), strftime('%m', order_date) in SQLite. This groups all rows sharing the same year-month together for aggregation.

2. What's the difference between COUNT(*) and COUNT(DISTINCT col)?

COUNT(*) counts every row including duplicates and NULLs. COUNT(DISTINCT col) counts only unique non-NULL values in that column. For order volume we use COUNT(DISTINCT order_id) to avoid counting duplicate entries of the same order.

3. How do you calculate monthly revenue?

Use SUM(amount) paired with GROUP BY year, month. This sums all order amounts within each month, giving the total revenue for that month.

4. What are aggregate functions in SQL?

Aggregate functions operate on a set of rows and return a single value. Common ones: SUM(), COUNT(), AVG(), MIN(), MAX(). They are used with GROUP BY to compute statistics per group.

5. How to handle NULLs in aggregates?

Most aggregate functions ignore NULLs automatically (e.g., SUM() skips NULL rows). To explicitly treat NULLs as zero, use COALESCE(amount, 0) inside the aggregate: SUM(COALESCE(amount, 0)).

6. What's the role of ORDER BY and GROUP BY?

GROUP BY defines how rows are combined into groups for aggregation (e.g., by month). ORDER BY sorts the final result set for readability or ranking (e.g., ascending by date, or descending by revenue).

7. How do you get the top 3 months by sales?

Group by year/month, apply SUM(amount), then sort with ORDER BY total_revenue DESC and restrict with LIMIT 3. Result: June (₹3,950) → March (₹2,800) → April (₹2,750).
